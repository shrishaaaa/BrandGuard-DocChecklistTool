What Makes a Document Professional?

✅ It IS about:

  • ✅ Consistency across every page
  • ✅ Clear visual hierarchy
  • ✅ Generous, deliberate whitespace
  • ✅ Effortless readability
  • ✅ Intentional use of color

❌ It is NOT about:

  • ❌ More content per page (cramming = chaos)
  • ❌ Decorative borders and effects (noise)
  • ❌ Multiple fonts or colors (inconsistency)
  • ❌ Justified text (looks neat, reads poorly)
  • ❌ Filling every inch of space (suffocating)

The Four Pillars

🌬️
Whitespace
Breathing room that reduces cognitive load and guides the eye to what matters.
📖
Readability
Optimal line length, alignment, and type scale for effortless reading.
📐
Hierarchy
Structure visible at a glance — headings, subheadings, body working together.
🎨
Brand Colors
Consistent palette that builds recognition and trust across every touchpoint.

Cramped vs. Spacious

Cramped → Overwhelm

Dense pages increase cognitive load. Readers skip, skim, or abandon. More content per page does not mean more professional — it signals "this was thrown together."

Spacious → Confidence

Well-spaced layouts signal control. The content breathes, so does the reader. Premium brands use whitespace as a trust signal — think Apple, not Craigslist.

"An 18-page report with good spacing is more professional than a 15-page report crammed together."

📐 Page Setup

Why This Matters

The foundation of every document. Wrong page size or margins immediately make a report look unprofessional or out of place when printed or shared.

The Rules

PropertyValue
FormatA4 Portrait (8.27" × 11.69")
Margins0.8 inches — all four sides (uniform)
OrientationPortrait
Background ColorWhite (#FFFFFF)

📋 How To Do It (Google Docs)

  1. Go to File → Page Setup
  2. Set Paper Size to A4 (8.27" × 11.69")
  3. Set all four margins to 0.8 inches
  4. Set Orientation to Portrait
  5. Set Page Color to White
  6. Click "Set as default" so all future docs use these settings

⚠️ Common Mistakes

  • Using Letter size instead of A4
  • Uneven margins (e.g., 1" top but 0.75" sides)
  • Landscape orientation for text-heavy reports
  • Colored or off-white backgrounds

🔤 Typography

Why This Matters

Typography creates the visual hierarchy that lets readers navigate a 20-page report without a table of contents. Size, weight, and color do the work. Without hierarchy, everything is a wall of text.

The Rules

Font: Geist — Used for all text across the document.

📋 How To Do It

  1. Install/add Geist font to your system or Google Docs
  2. Set up your document styles: Format → Paragraph Styles → Update each heading level to match the table above
  3. Always use the defined heading styles rather than manually changing font size
  4. Never use font sizes outside this scale — if something doesn't fit, it belongs in an existing level

⚠️ Common Mistakes

  • Using multiple font families in one document
  • Making text bigger/bolder instead of using the correct heading level
  • Using font sizes not in the scale (e.g., 16px, 20px, 24px)
  • Forgetting to set heading colors (leaving them all black)

📏 Layout & Spacing

Why This Matters

84% of users prefer clean, simple designs. Whitespace is not empty — it reduces cognitive load, creates hierarchy, and signals confidence. An 18-page report with good spacing is more professional than a 15-page report crammed together.

The Rules

  • Two-column layout: Left column (~30%) for navigation/labels, Right column (~70%) for content
  • Optimal line length: 50–75 characters per line — the two-column layout automatically enforces this
  • Text alignment: Left-aligned. ALWAYS. Never justified.
  • Whitespace: Let pages breathe. Don't fill every inch.
Why Left-Aligned Over Justified:
  • Left-aligned text leads to better task performance in visual search
  • Justified text creates irregular word spacing that increases reading time
  • WCAG 1.4.8 accessibility guidelines recommend avoiding justified text

📋 How To Do It

  1. Set up a two-column table (borderless): left column 30%, right column 70%
  2. Set text alignment to Left (Format → Align → Left)
  3. Never use Justify alignment — even if it "looks neater"
  4. Add paragraph spacing between sections (use space-after, not empty lines)
  5. If a page feels cramped, move content to the next page — don't shrink fonts or reduce spacing

⚠️ Common Mistakes

  • Using full-width single-column text (line length too long)
  • Justified text alignment
  • Cramming content to avoid adding pages
  • Zero spacing between sections

📊 Tables

Why This Matters

Tables present data. Poorly formatted tables make data unreadable and make the document look amateur. Consistent table styling across a report builds a professional rhythm.

The Rules

PropertyValue
Font Size10px
Line SpacingSingle
Paragraph SpacingNone
Header Text ColorWhite
Header Bg (Option A)Brand Blue #015CE6
Header Bg (Option B)Black #121212
Body Text ColorBlack #121212
Body Bg (Option A)Light Blue #E8F1FF
Body Bg (Option B)Light Grey #F0F4F9
Border Weight3pt
Border ColorWhite (acts as cell separator)
Cell Padding0.1 inches
StyleInline, Left-aligned
Body Cell AlignmentTop
Header Cell AlignmentMiddle

📋 How To Do It

  1. Insert table → set font to Geist, size 10px
  2. Select header row → background #015CE6 (or #121212), text color White, alignment Middle
  3. Select body rows → background #E8F1FF (or #F0F4F9), text color #121212, alignment Top
  4. Set all borders to 3pt, color White
  5. Set cell padding to 0.1 inches (Table Properties → Cell → Cell padding)
  6. Align the table to the left

⚠️ Common Mistakes

  • Default Google Docs/Word table styling (thin black borders, no background colors)
  • Inconsistent header colors across tables in the same document
  • Body text not aligned to top of cell
  • Missing cell padding (text touching cell borders)

📄 Cover Page

Why This Matters

50ms — that's how long readers take to form a visual opinion. 94% of first impressions are design-related. 75% judge credibility based on visual design. The cover page decides the entire perception of your report.

The Rules

  • Distinct cover page with Brand Blue gradient/background
  • Title: Large, white, prominent text (34px, White)
  • Subtitle: Supporting text (18px, White)
  • SecurityPal AI branding/logo present
  • Cover page footer is different from body page footer

📋 How To Do It

  1. Use the approved cover page template (created in Canva)
  2. Or create manually: Set page background to Brand Blue (#015CE6)
  3. Add title in 34px White, Geist font
  4. Add subtitle in 18px White, Geist font
  5. Place SecurityPal AI logo in the top-left area
  6. Don't add body content to the cover page — it's for branding only

⚠️ Common Mistakes

  • No cover page (jumping straight into content)
  • White/plain cover page with just a title
  • Using off-brand colors or gradients
  • Too much text on the cover page
  • Missing company branding

🔻 Footer

Why This Matters

The footer provides consistency and navigation across every page. It anchors the document and reinforces branding on every page the reader sees.

The Rules

  • Footer text uses Heading 5 style: 8px, Grey/Brand Blue
  • Same footer appears on every body page
  • Cover page has a different, pre-designed footer
  • Footer is editable via double-click in Google Docs

📋 How To Do It

  1. Double-click the footer area to enter editing mode
  2. Set the font to Geist, 8px
  3. Set the color to Grey (#828282) or Brand Blue (#015CE6)
  4. Add standard footer content (company name, document title, page number, confidentiality notice)
  5. The cover page footer should be set separately and should be different from body pages

⚠️ Common Mistakes

  • No footer at all
  • Footer text too large (using normal body text size)
  • Different footers on different body pages (inconsistency)
  • Same footer on cover page as body pages

📋 How To Do It

  1. Open Canva and use the SecurityPal AI template (if available)
  2. Set your primary chart color to #015CE6
  3. Use only the approved shades of blue for secondary data series
  4. Place chart titles/names below the chart, not above
  5. Export as high-resolution image and insert into the document
  6. Ensure the chart doesn't stretch or distort when inserted

⚠️ Common Mistakes

  • Using default Excel/Sheets chart colors (rainbow palette)
  • Chart names above instead of below
  • Using non-blue colors for data series
  • Low-resolution chart exports
  • Charts that are too small or too large for the page

🚫 What NOT To Do

Why This Matters

Knowing what to avoid is as important as knowing what to do. These are the most common mistakes that make documents look unprofessional.

🚫 Cramming Content Per Page

More content per page does NOT mean more professional. Cramming increases cognitive load. Readers skip, skim, or abandon. Dense pages signal "this was thrown together."

🚫 Decorative Borders and Effects

Drop shadows, decorative borders, gradient fills on text boxes — these are noise, not design. Professional documents are intentional, not decorated.

🚫 Multiple Fonts or Too Many Colors

Stick to Geist. Stick to the 6-color palette. Every additional font or color weakens consistency and makes the document look chaotic.

🚫 Justified Text

It looks neat at first glance, but justified text creates uneven word spacing. This increases reading time and reduces comprehension. Left-aligned text is faster to read, more accessible, and recommended by WCAG.

🚫 Filling Every Inch of Space

Whitespace is not wasted space. It is working space. It reduces cognitive load, creates hierarchy, and signals confidence. Think Apple, not Craigslist.

✅ The Rule of Thumb

If you're unsure whether to add more content to a page or start a new page — start a new page. Let it breathe.
